PROJECT TRACKING SYSTEM (MANDATORY)
==================================================

Create a permanent project tracking system using Markdown files.

The following files are required:

1. PROJECT_STATUS.md
2. PROJECT_PROGRESS.md
3. PROJECT_TODO.md
4. CHANGELOG.md
5. DECISIONS.md

==================================================
DEVELOPMENT RULES
==================================================

After finishing ANY feature:

1. Update PROJECT_STATUS.md first.
2. Update PROJECT_PROGRESS.md.
3. Move completed tasks from PROJECT_TODO.md.
4. Add new discovered tasks.
5. Update CHANGELOG.md.

Never finish a coding session without updating these files.

Before starting new work:

Read PROJECT_STATUS.md first.

Always continue from the last saved state.

Never assume previous work is complete unless documented.

==================================================
FINAL REQUIREMENT
==================================================

At the beginning of every development session:

Read:

PROJECT_STATUS.md

PROJECT_TODO.md

CHANGELOG.md


Then summarize:

- What has been completed.
- What is currently being built.
- What remains.
- The next recommended action.

Do not start coding before understanding the current project state.
